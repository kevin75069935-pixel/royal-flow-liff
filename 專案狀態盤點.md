## 1. 專案定位

此專案是御澤禮賓商務的 LIFF 預約系統，負責客戶預約、初步報價、訂單寫入、付款資訊、LINE 查詢、司機派遣、行前提醒與評價邀請。

主責歸屬為 `訂單管家`，但會串接：

| 串接員工 | 串接內容 |
| --- | --- |
| 禮賓客服 | LINE 官方帳號回覆、關鍵字導流、客服轉接 |
| 報價助理 | Google Routes 距離估價、車型與加值服務報價 |
| 回訪行銷員 | 服務完成後評價邀請與回訪 |

## 4. 待部署項目

1. 將 `apps-script/Code.gs` 貼到 Google Apps Script。
2. 綁定正式訂單 Google Sheet。
3. 設定 Apps Script Script Properties。
4. 執行或呼叫 `setupPricingSheets()` 建立報價設定分頁。
5. 重新部署 Apps Script Web App 並取得 `/exec` URL。
6. 將專案部署到 Cloudflare Pages。
7. 在 Cloudflare Pages 設定環境變數與 KV binding。
8. 到 LINE Developers 設定 LIFF Endpoint URL。
9. 到 LINE Developers 設定 Messaging API Webhook URL：

```text
https://你的 Cloudflare Pages 網址/api/line/webhook
```

10. 準備並上架 LINE Rich Menu 圖片與 `line-rich-menu.json`。

## 6. 目前風險與注意事項

- 專案目前沒有 Git remote；若要讓 Cloudflare Pages 自動部署，需要先連到 GitHub 或用 Cloudflare Pages 直接上傳。
- `index.html` 內的 LIFF ID 目前寫死為 `2010083928-gA1wvCJ0`，若更換 LIFF ID，需要同步更新前端、README、Rich Menu 設定。
- `README.md` 與部署指南目前仍使用 `royal-flow-liff.pages.dev` 作為範例網域；若改用正式網域，需要同步更新 `ALLOWED_HOSTS`、LINE LIFF Endpoint、Webhook URL 與 `PUBLIC_BASE_URL`。
- Apps Script 修改後必須重新部署 Web App 新版本，否則 Cloudflare 仍會呼叫舊版。
- 自動報價依 Google Routes 與試算表規則計算，正式報價仍建議保留人工確認。
- 付款狀態目前以訂單資料與人工/同步更新為主，尚未確認是否已串接正式金流自動對帳。
